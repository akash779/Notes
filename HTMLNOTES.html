WEB FORM

<form>: The container tag for a web form. It establishes a form to collect user input and send it to the server for processing.
- Attributes:
- action: Specifies the URL where the form data should be submitted.
- method: Specifies the HTTP method to be used when submitting the form (GET or POST).
<input>: Creates various types of input fields.
- Attributes:
- type: Specifies the type of input field (e.g., text, email, password, checkbox, radio, etc.).
- name: Provides a name for the input field, which will be used to identify the input data when the form is submitted.
- id: Assigns a unique identifier for the input field, useful for associating the field with a <label> or for scripting purposes.
- value: Sets the initial value of the input field.
- required: Specifies that the field must be filled out before submitting the form.
<textarea>: Creates a multi-line text input field.
- Attributes:
- name: Provides a name for the input field, used to identify the input data when the form is submitted.
- rows: Specifies the number of visible rows for the textarea.
<select>: Creates a dropdown list or a list box.
- Attributes:
- name: Provides a name for the select element, used to identify the selected option when the form is submitted.
<option>: Defines an option within a <select> element.
- Attributes:
- value: Specifies the value associated with the option.
- selected: Specifies that the option should be pre-selected when the page loads.
<label>: Represents a text label for an input field.
- Attributes:
- for: Specifies which form element the label is associated with by referencing the element's id attribute.
<button>: Creates a button within a form.
- Attributes:
- type: Specifies the behavior of the button (submit, reset, or button).
- name: Provides a name for the button, used to identify the button's value when the form is submitted.
<fieldset>: Groups related form elements together.
- Attributes:
- disabled: Disables all form elements within the fieldset.
<legend>: Provides a caption or title for a <fieldset> element.
*These are the fundamental tags and attributes used in web forms. You can use them in combination to build complex and interactive forms to collect user input on your website.

CSS

CSS typically refers to the Cascading Style Sheets language used for styling web pages. However, it's not possible to provide a meaningful representation of the entire CSS language in just two lines. CSS encompasses a wide range of selectors, properties, and values that are used to style the various elements of a web page.

Ways to link css

CSS Types - Inline, Internal, and External

Inline CSS

Inline CSS is applied directly to individual HTML elements using the style attribute.

<p style="color: blue; font-size: 16px;">This is a blue paragraph with font size 16px.</p>

Internal CSS

Internal CSS is defined within the <style> tags in the <head> section of the HTML document.

<head> <style> p { color: blue; font-size: 16px; } </style> </head>

External CSS

External CSS is stored in a separate CSS file and linked to the HTML document using the <link> tag.

<head> <link rel="stylesheet" type="text/css" href="styles.css"> </head>


CSS Selectors - Types and Syntax

Element Selector

The element selector targets specific HTML elements.

p { /* Styles for <p> elements */ }

Class Selector

The class selector targets elements with a specific class attribute.

.code { /* Styles for elements with class="code" */ }

ID Selector

The ID selector targets a single element with a specific ID attribute.

#myElement { /* Styles for element with id="myElement" */ }

Descendant Selector

The descendant selector targets elements that are descendants of a specific parent element.

.parentElement code { /* Styles for <code> elements within .parentElement */ }

Universal Selector

The universal selector targets all elements in the HTML document.

*

Attribute Selector

The attribute selector targets elements based on their attribute values.

[type="text"] { /* Styles for elements with type="text" attribute */ }

Pseudo-Selectors

Pseudo-selectors target elements based on their state or position within the document.

p:first-child { /* Styles for the first <p> element within its parent */ }

a:hover { /* Styles for <a> elements when hovered *}



CSS Basic Properties

Background

The background properties allow you to customize the background of an element.

  • background-color: sets the background color.
  • background-image: sets the background image.
  • background-attachment: sets whether the background scrolls with the content or remains fixed.
  • background-repeat: sets how the background image is repeated.
  • background-size: sets the size of the background image.

Example:

        body {
            background-color: #f5f5f5;
            background-image: url('background.jpg');
            background-attachment: fixed;
            background-repeat: no-repeat;
            background-size: cover;
        }

Fonts

The font properties allow you to control the appearance of text.

  • font-family: sets the font family.
  • font-style: sets the font style (e.g., italic).
  • font-size: sets the font size.
  • font-weight: sets the font weight (e.g., bold).

Example:

        p {
            font-family: 'Arial', sans-serif;
            font-style: italic;
            font-size: 18px;
            font-weight: bold;
        }

Text Formatting

The text formatting properties allow you to control the appearance of text content.

  • color: sets the text color.
  • text-align: aligns the text horizontally.
  • text-align-last: aligns the last line of text in a block.

1. **Flexbox Overview**:
   - Flexible Box Layout (Flexbox) is a CSS layout model.
   - Designed for flexible and efficient arrangement of elements.
   - Controls element alignment and distribution in containers.

2. **Main Axis and Cross Axis**:
   - Flexbox operates along main and cross axes.
   - Main axis: primary direction; cross axis: perpendicular direction.

3. **Flex Direction**:
   - Property defines main axis direction.
   - Options: row, row-reverse, column, column-reverse.

4. **Flex Items**:
   - Child elements of flex container.
   - Laid out using Flexbox rules.

5. **Justify Content**:
   - Controls item distribution along main axis.
   - Options: space between, start, center, end, space evenly.

6. **Align Items**:
   - Determines item alignment along cross axis.
   - Controls vertical alignment.

7. **Align Self**:
   - Overrides parent container's align-items.
   - Applies to individual flex items.

8. **Flex Wrap**:
   - Specifies wrapping of flex items when container width is exceeded.

9. **Flex Grow, Flex Shrink, Flex Basis**:
   - Properties control item sizing and distribution.
   - Manage available space within the container.

10. **Benefits of Flexbox**:
    - Simplifies responsive layouts and alignment.
    - Eliminates complex float and positioning techniques.
    - Suited for one-dimensional layouts (rows/columns).
    - Handles varying content sizes and dynamic layouts.

11. **CSS Grid Comparison**:
    - CSS Grid excels at two-dimensional layouts.
    - Flexbox is one-dimensional; Grid is two-dimensional.
    - Choose based on layout requirements.
    - Can use both Flexbox and Grid together.
